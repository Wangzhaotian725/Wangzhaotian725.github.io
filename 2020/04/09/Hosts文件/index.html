Hosts文件

发表于 2020-04-09 分类于 计算机网络

越来越长时间地游走在国外的网页中，曾经使用过VPN的我现在竟然觉得缺少梯子在互联网的世界寸步难行（原来的梯子在2019年国庆由于众所周知的原因失效了……）。然而家境贫穷迫于无奈退而求其次采用了修改hosts文件的方式，在一定程度上提升访问国外服务器的速度。

Hosts文件的工作方式

在过去引入DNS域名系统之前，计算机从Hosts文件中的IP地址和域名的映射关系通过FTP协议接入互联网。

在现在计算机在提高DNS请求之前依然会检查Hosts文件中的IP地址和域名之间的映射关系。如果有，则直接访问该网络位置；若没有，则向已知的DNS提交域名解析请求。

电脑中的Hosts文件的使用方法

Windows系统若系统盘为C盘，则电脑中的Hosts文件储存位置为：

C:\Windows\System32\drivers\etc

Host文件没有后缀名，可以用记事本打开，这里推荐使用notepad++。

其中默认的开头为：

# Copyright (c) 1993-2009 Microsoft Corp.
#
# This is a sample HOSTS file used by Microsoft TCP/IP for Windows.
#
# This file contains the mappings of IP addresses to host names. Each
# entry should be kept on an individual line. The IP address should
# be placed in the first column followed by the corresponding host name.
# The IP address and the host name should be separated by at least one
# space.
#
# Additionally, comments (such as these) may be inserted on individual
# lines or following the machine name denoted by a ‘#’ symbol.
#
# For example:
#
# 102.54.94.97 rhino.acme.com # source server
# 38.25.63.10 x.acme.com # x client host
# localhost name resolution is handled within DNS itself.

如果怀疑Hosts文件被病毒恶意篡改，可以清空文件内容将以上粘贴到文件中。不过#号后内容皆为注释，没有实际意义。

添加IP地址与域名的方式如下：

192.30.253.112 github.com
192.30.253.119 gist.github.com

注意IP地址与域名映射关系是否准确，可以时不时ping一下网页更新IP地址。

同时Hosts文件也可以用来屏蔽域名，例如：

# 127.0.0.1 baidu.com

其中127.0.0.1为本地地址

在GitHub上了解更多内容：https://github.com/lennylxx/ipv6-hosts
